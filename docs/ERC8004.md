# ERC-8004 Integration Guide

## Registration Flow

1. **Build metadata** — Agent registration JSON per EIP-8004
2. **Upload to IPFS** — Pinata JWT or local data URI fallback
3. **Call `register(agentURI)`** — IdentityRegistry mints agent NFT
4. **Update URI** — `setAgentURI(agentId, fullMetadataUri)` with complete registration file
5. **Store Agent ID** — Save to PostgreSQL and `.env`

## Agent Metadata Schema

```json
{
  "type": "https://eips.ethereum.org/EIPS/eip-8004#registration-v1",
  "name": "ARC Credit Agent",
  "version": "1.0.0",
  "description": "AI-powered on-chain credit scoring agent.",
  "capabilities": ["wallet_analysis", "credit_scoring", "risk_assessment"],
  "services": [
    { "name": "REST API", "endpoint": "http://localhost:3001/api/v1" }
  ],
  "registrations": [
    {
      "agentId": 1,
      "agentRegistry": "eip155:5042002:0x8004A818BFB912233c491871b3d84c89A494BD9e"
    }
  ],
  "supportedTrust": ["reputation", "validation"]
}
```

## Reputation

Validators call `giveFeedback()` on ReputationRegistry. Agent owners cannot self-review.

Tags used:
- `tag1`: `credit_scoring`
- `tag2`: `validation`

## Validation

1. Agent owner calls `validationRequest(validator, agentId, requestURI, requestHash)`
2. Validator reviews off-chain payload at `requestURI`
3. Validator calls `validationResponse(requestHash, response, responseURI, responseHash, tag)`
4. `response`: 1 = approved, 0 = rejected

## Upgrading Metadata

```typescript
await identityService.upgradeMetadata(agentId);
```

Calls `setAgentURI` with updated version and re-uploads to IPFS.
